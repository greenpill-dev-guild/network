# Self-Hosting and Git Sovereignty Evaluation

## Required Outputs

- a clear recommendation matrix
- explicit go or no-go criteria
- exact prerequisites for Mac Mini hosting
- exact reasons to keep some surfaces managed

## Proof Expectations

The finished decision pack should be able to answer:

- why Git is or is not the first move
- what conditions must be true before home-hosting the public website
- what rollback path exists if a public cutover ever happens
- why backend-class services remain managed or when that answer changes

## Review Bar

- recommendations stay grounded in the current repo and `.plans/active/public-website-design-implementation/artifacts/v2/` architecture direction
- sovereignty benefits are separated from operational burden
- staging proof is required before any production recommendation
- the plan does not quietly expand into full-stack self-hosting without new evidence
