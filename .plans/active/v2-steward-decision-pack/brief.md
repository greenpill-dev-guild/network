# V2 Steward Decision Pack

**Stage**: `active`

## Why This Exists

The workshop synthesis produced enough alignment to move toward V2 execution, but a few narrative and visual-content decisions still need steward input before the homepage and first public routes should be treated as locked.

This hub keeps that decision process in `.plans/`, while `docs/v2/` remains the product and architecture source of truth.

## Desired Outcome

Stewards receive a short, readable brief plus a small Telegram question set. Their responses should resolve the open choices that shape the first V2 implementation pass:

- what a newcomer should understand first
- what proof should show that Greenpill is active now
- how Greenpill Garden should appear in the first release
- which chapters, guilds or pods, and builder projects should be featured first

The outcome is a concise response summary that can update the canonical V2 docs, the AI build manifest, and the first implementation handoff.

## Non-Goals

- revisiting the locked hosting, auth, content, or workspace architecture
- producing a full research report for stewards
- turning every UI preference into a poll
- starting implementation before the key steward-facing narrative decisions are summarized
