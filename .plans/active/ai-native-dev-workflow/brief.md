# AI-Native Developer Workflow Brief

## Goal

Raise Greenpill Network's AI-assisted workflow into a consistently provable AI-native development system over six weeks. The operating target is reliable delegation with less cognitive debt: every active feature has explicit intent, assigned agent roles, verification evidence, human judgment callouts, a run ledger, a workflow scorecard, adversarial review, and a closeout gate.

## Why Now

The repo already has strong agent-assisted practice. The next improvement is making agent work measurable and repeatable across feature planning, implementation, review, and closeout without creating a parallel source of truth.

## Repo Focus

Focus on public/private data boundaries, public-safe map and content contracts, Directus/Postgres/Keystatic separation, website browser proof, and deployment evidence.

## Six-Week Outcome

- Week 1: this native plan hub exists and validates.
- Week 2: one active feature records an agent run ledger.
- Week 3: one recent feature has a workflow scorecard baseline.
- Week 4: one adversarial review pass is captured read-only.
- Week 5: closeout gate language is wired into the repo's existing validation habits.
- Week 6: a retrospective updates only the guidance that proved useful.
