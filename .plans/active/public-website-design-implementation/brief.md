# Public Website Design Implementation

**Stage**: `active`

## Why This Exists

The repo is moving from scaffold and infrastructure setup into the public website design implementation pass. The prior `docs/` folder mixed temporary strategy, research, prompts, and generated design handoffs in a way that made short-lived planning material look permanent.

This hub is now the execution home for the public website redesign. It keeps the high-fidelity design package, research inputs, prompts, and V2 planning artifacts beside the implementation plan they support.

## Desired Outcome

- Implement the public Astro + Keystatic website from the high-fidelity design artifacts.
- Keep research and design references available while the work is active, then archive them with this hub when the implementation closes.
- Avoid recreating a root `docs/` knowledge base for material that is expected to change or go stale.

## Non-Goals

- Do not implement private workspace, steward dashboard, or authenticated member states in this hub.
- Do not restore root-level `docs/` as a parallel source of truth.
- Do not treat the prototype JSX in `artifacts/hifi/` as production source code.
